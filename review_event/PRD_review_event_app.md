# PRD: 네이버 리뷰 이벤트 참여 확인 앱

## 1. 개요

### 배경
드림캐쳐스 공유오피스는 매니저 비상주 시간(평일 오후 2시 이후, 주말 전일)이 대부분을 차지한다.
네이버 영수증 리뷰 활성화를 위해 냉장고 앞 패드를 통한 셀프 이벤트 참여 시스템이 필요하다.
참여자는 비타1000 음료를 자율 수령하며, 시스템은 참여자의 자기 구속력(Commitment)을 활용해 리뷰 이행률을 높인다.

### 목표
- 매니저 개입 없이 이벤트가 자동 운영되는 무인 참여 시스템 구축
- 네이버 리뷰 수 증가를 통한 플랫폼 노출 확대
- 최소한의 정보 수집으로 참여 패턴 분석 및 어뷰징 억제

## 3-1. 브랜드 컬러 가이드 (드림캐쳐스)

로고 이미지 시각 분석 기준. 정밀 수치는 로고 원본 파일에서 스포이드로 재확인 권장.

| 역할 | 이름 | HEX | 용도 |
|------|------|-----|------|
| Primary | Forest Green | `#1E5631` | 왕관 아이콘, CATCHERS 텍스트 색상. 주요 버튼 배경, 헤더, 강조 요소 |
| Dark BG | Deep Forest Green | `#2D5A3D` | 다크버전 로고 배경색. 시작화면 상단 헤더 배경 등 포인트 영역에 활용 |
| Secondary | Charcoal Gray | `#404040` | DREAM 텍스트 색상. 본문 텍스트, 라벨, 설명 문구 |
| Background | White | `#FFFFFF` | 전체 화면 기본 배경 |
| Button Inactive | Light Gray | `#D1D5DB` | 약속 체크 전 비활성 버튼 |
| Accent | Light Green | `#D6EAD8` | 완료 화면 배경, 성공 상태 강조 |
| White (on green) | White | `#FFFFFF` | 다크그린 배경 위 텍스트·아이콘 |

**컬러 시스템 적용 원칙:**
- 흰 배경 + `#1E5631` 버튼이 기본 조합 — 가독성과 브랜드 일관성 동시 확보
- 시작 화면 헤더 영역은 `#2D5A3D` 배경 + 흰 텍스트로 다크버전 로고와 통일감
- 비활성 버튼: `#D1D5DB` → 약속 체크 시 `#1E5631` 으로 전환 (명확한 시각적 피드백)
- 완료 화면 전체 배경: `#D6EAD8` — 성공·긍정 감정 유도
- 텍스트 색상: `#404040` 차콜그레이 — 순수 블랙보다 눈의 피로 감소

## 4. 사용자 플로우

```
[1. 시작 화면]  이벤트 안내 + 음료 이미지 + 참여 버튼
    ↓
[2. 입력 화면]  정보 입력 + 약속 체크
    ↓  (중복 감지 시 → 안내 팝업으로 분기)
[3. 완료 화면]  감사 인사 + 네이버 리뷰 QR코드
    ↓
[시작 화면으로 자동 복귀 — 10초]
```

> 3단계 고정. 유형 구분(입주사/라운지) 탭 제거 — 입력 필드를 한 화면에 나열하고 해당 항목만 채우면 된다.

## 5. 화면별 상세 설계

### 5-1. 시작 화면 (이벤트 안내)

```
┌─────────────────────────────────────┐
│                              [관리자] │  ← 우상단, 소형 텍스트
│                                      │
│   네이버 리뷰 쓰고                   │
│   시원하게 충전하세요! 🎁            │  ← 32px 이상, 상단 고정
│                                      │
│  ┌──────────────────────────────┐    │
│  │   [비타1000 음료 이미지]     │    │  ← 화면 중앙, 크게
│  └──────────────────────────────┘    │
│                                      │
│  ┌──────────────────────────────┐    │
│  │   ✅  참여하기               │    │  ← 버튼 높이 80px+, 강한 포인트색
│  └──────────────────────────────┘    │
│                                      │
│    하루에 한 번 참여 가능합니다      │  ← 작은 글씨, 하단
│                                      │
└─────────────────────────────────────┘
```

**이미지 소스:**
- URL: `https://link.coupang.com/a/ewgF3C?abTestInfo=100412_group_A`
- 해당 쿠팡 링크에서 비타1000 제품 이미지를 크롤링 또는 스크린샷하여 앱 내 에셋으로 저장
- 저작권 이슈 회피를 위해 직접 임베드하지 않고 로컬 이미지 파일로 변환 사용 권장
- 이미지 비율: 세로형 카드 이미지 (3:4 또는 정방형)

**규칙:**
- 버튼은 화면 전체에서 1개만 존재.
- "하루 한 번 참여 가능" 문구는 버튼 아래 고정 노출 — 중복 시도 선제 차단.
- 우상단 `[관리자]` 버튼: 텍스트 소형, PIN 4자리 입력 후 대시보드 진입.

### 5-2. 입력 화면 (정보 입력)

유형 탭 구분 없이 한 화면에 모든 필드를 나열. **아래 중 해당하는 항목만 채우면 된다.**

```
┌─────────────────────────────────────┐
│                                      │
│  정보를 입력해 주세요                │  ← 안내 문구
│  (해당 항목만 입력하면 됩니다)       │
│                                      │
│  네이버 아이디                        │
│  ┌──────────────────────────────┐    │
│  │                              │    │
│  └──────────────────────────────┘    │
│                                      │
│  호실 번호  (입주사)                  │
│  ┌──────────────────────────────┐    │
│  │                              │    │  ← 숫자 키패드 자동 표시
│  └──────────────────────────────┘    │
│                                      │
│  성명 앞 2글자  (라운지)              │
│  ┌──────────────────────────────┐    │
│  │                              │    │  ← 2자 입력 시 자동 잠금
│  └──────────────────────────────┘    │
│                                      │
│  [ ☐ ]  리뷰를 쓰겠습니다           │  ← 체크 필수
│                                      │
│  ┌──────────────────────────────┐    │
│  │   확인   →                   │    │  ← 버튼 80px+, 체크 전 비활성
│  └──────────────────────────────┘    │
│                                      │
│  드림캐쳐스는 여러분의 신뢰를         │
│  믿습니다.                           │  ← 소형, 하단
└─────────────────────────────────────┘
```

**입력 유효성 규칙:**
- 3개 필드 중 **최소 1개 이상** 입력해야 `[확인]` 버튼 활성화
- 약속 체크박스는 **필수** (미체크 시 버튼 비활성 유지)
- 호실 번호는 숫자만 허용, 성명 앞 2글자는 최대 2자 제한
- 네이버 아이디는 영문+숫자+특수문자 허용, 공백 불가

---

### 5-3. 중복 참여 안내 팝업

입력 완료 후 당일 중복이 감지되면, 완료 화면으로 넘어가지 않고 팝업 표시.

```
┌────────────────────────────┐
│                            │
│   오늘은 이미 참여하셨어요 │
│                            │
│   내일 또 만나요! 😊       │
│                            │
│        [ 확인 ]            │
└────────────────────────────┘
```

- 경고 느낌 없이 친근한 톤 유지.
- `[확인]` 버튼 누르면 시작 화면으로 복귀.

### 5-4. 완료 화면 (감사 + 네이버 리뷰 연결)

```
┌─────────────────────────────────────┐
│                                      │
│   🎉 감사합니다!                     │  ← 큰 글씨, 팡파르 애니메이션
│   냉장고에서 비타1000을 꺼내 드세요  │
│                                      │
│  ┌──────────────────────────────┐    │
│  │                              │    │
│  │       [ QR 코드 ]            │    │  ← 중앙, 크게 (최소 200×200px)
│  │                              │    │
│  └──────────────────────────────┘    │
│                                      │
│   📱 QR코드를 찍으면                 │
│      네이버 리뷰 페이지로 이동해요   │  ← 28px
│                                      │
│   참여번호  #0042                    │  ← 소형, 하단
│   (10초 후 자동으로 돌아갑니다)      │
│                                      │
└─────────────────────────────────────┘
```

**QR코드 상세:**
- 연결 URL: `https://m.place.naver.com/my/review`
- QR코드는 앱 빌드 시 정적 생성 (빌드 타임에 qrcode 라이브러리로 생성하여 이미지로 저장)
- 크기: 최소 200×200px, 여백(Quiet Zone) 충분히 확보
- QR 아래 안내 문구: "QR코드를 찍으면 네이버 리뷰 페이지로 이동해요"

**기타:**
- 팡파르 애니메이션 (폭죽 또는 별 이펙트) 진입 시 1회 재생.
- 별도 버튼 없이 10초 카운트다운 후 시작 화면 자동 복귀.
- 참여 번호는 전체 누적 번호 표시 (심리적 완결감).

## 6. 데이터 모델

### 참여 기록 테이블 (`review_participations`)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | INTEGER PK | 자동 증가 |
| `participated_at` | DATETIME | 참여 일시 |
| `naver_id` | VARCHAR(50) NULL | 네이버 아이디 (선택) |
| `room_number` | VARCHAR(10) NULL | 호실 번호 (입주사, 선택) |
| `name_prefix` | VARCHAR(2) NULL | 성명 앞 2글자 (라운지, 선택) |
| `committed` | BOOLEAN | 약속 체크박스 동의 여부 (필수) |

> **제약 조건:** `naver_id`, `room_number`, `name_prefix` 중 최소 1개는 NOT NULL — DB 레벨 CHECK 제약 또는 API 레이어에서 검증.
> `user_type` 컬럼 제거 — 필드 입력 여부로 유형을 유추할 수 있어 별도 구분 불필요.

## 10-1. PWA 설정 명세

전체화면 실행을 위해 반드시 아래 설정이 포함되어야 한다.

### manifest.json 핵심 항목

```json
{
  "name": "드림캐쳐스 리뷰 이벤트",
  "short_name": "리뷰참여",
  "display": "fullscreen",
  "orientation": "portrait",
  "theme_color": "#1E5631",
  "background_color": "#FFFFFF",
  "start_url": "/",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

> `"display": "fullscreen"` — URL바, 상태바, 안드로이드 네비게이션 바(뒤로/홈/최근앱)까지 모두 숨긴다.
> `"display": "standalone"` 은 상태바가 남으므로 사용하지 않는다.

### viewport meta 태그

```html
<meta name="viewport"
  content="width=device-width, initial-scale=1.0,
           maximum-scale=1.0, user-scalable=no" />
```

## 10-2. 디바이스 초기 설정 가이드 (운영자용)

기기 1대당 최초 1회 작업. 이후 전원 켜면 자동으로 전체화면 앱으로 실행됨.

```
1. Android 기기에서 Chrome으로 앱 URL 접속
2. Chrome 우상단 메뉴(⋮) → [홈 화면에 추가] 탭
3. 이름 확인 후 [추가] 선택
4. 홈 화면에 생성된 아이콘 실행 → URL바 없는 전체화면으로 앱 시작
5. Android 설정 → 디스플레이 → 화면 자동 꺼짐: [끄지 않음] 또는 최대값 설정
6. (선택) Android 설정 → 접근성 → 터치 잠금 앱 설정으로 홈버튼 비활성화
```

> 기기가 여러 대라도 동일한 URL에 접속하면 동일하게 설정 가능.
> 앱 업데이트 시 서버 배포만으로 모든 기기에 자동 반영됨 (네이티브 앱 불필요).
