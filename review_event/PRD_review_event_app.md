# PRD: 네이버 리뷰 이벤트 참여 확인 앱

## 1. 개요

### 배경
드림캐쳐스 공유오피스는 매니저 비상주 시간(평일 오후 2시 이후, 주말 전일)이 대부분을 차지한다.
네이버 영수증 리뷰 활성화를 위해 냉장고 앞 패드를 통한 셀프 이벤트 참여 시스템이 필요하다.
참여자는 비타1000 음료를 자율 수령하며, 시스템은 참여자의 자기 구속력(Commitment)을 활용해 리뷰 이행률을 높인다.

### 목표
- 매니저 개입 없이 이벤트가 자동 운영되는 무인 참여 시스템 구축
- 네이버 리뷰 수 증가를 통한 플랫폼 노출 확대
- 최소한의 정보 수집으로 참여 패턴 분석 및 어뷰징 억제

---

## 2. 사용자 및 운영 환경

| 항목 | 내용 |
|------|------|
| 사용 기기 | 냉장고 앞 설치 태블릿 패드 |
| 사용자 유형 | 드림캐쳐스 계약자(호실 입주사), 라운지 이용자 |
| 운영 시간 | 24시간 (매니저 상주: 평일 10:00~14:00) |
| 이벤트 경품 | 냉장고 내 비타1000 음료 (1인 1병) |
| 연계 플랫폼 | 네이버 지도 영수증 리뷰 |

---

## 3. 사용자 플로우

```
[시작 화면]
    ↓
[유형 선택] → 계약자(호실) / 라운지 이용자
    ↓
[정보 입력]
    ↓
[약속 확인 (체크박스)]
    ↓
[완료 화면 + 이벤트 당첨 확인]
```

---

## 4. 화면별 상세 설계

### 4-1. 시작 화면

- **메인 버튼 1개**: `리뷰 작성하고 비타민 충전하기`
- 버튼 아래 안내 문구:
  > "네이버 영수증 리뷰를 작성(또는 예정)하셨나요?  
  > 시원한 비타1000으로 에너지 충전하세요!"
- 우측 상단: 운영자용 관리 진입 아이콘 (PIN 잠금)

---

### 4-2. 유형 선택 화면

두 개의 카드형 버튼:

| 버튼 | 설명 |
|------|------|
| 드림캐쳐스 입주사 | 계약 호실 보유자 |
| 라운지 이용자 | 1일권 / 자유석 이용자 |

---

### 4-3. 정보 입력 화면

#### 공통 수집 항목
| 필드 | 입력 형식 | 비고 |
|------|-----------|------|
| 네이버 아이디 | 텍스트 | 필수 |

#### 입주사 추가 항목
| 필드 | 입력 형식 | 비고 |
|------|-----------|------|
| 호실 번호 | 숫자 선택 또는 직접 입력 | 필수 |

#### 라운지 이용자 추가 항목
| 필드 | 입력 형식 | 비고 |
|------|-----------|------|
| 이름 앞 2글자 | 텍스트 (2자 제한) | 필수 |

> **설계 의도**: 전화번호나 실명 전체를 받지 않아 심리적 부담을 낮추되,
> 동일인 중복 참여 억제에 충분한 식별자를 확보한다.

---

### 4-4. 약속 확인 화면 (핵심 UX)

**체크박스 문구:**
> [ ] 네이버 영수증 리뷰를 작성했거나, 오늘 중으로 작성할 것을 약속합니다.

- 체크하지 않으면 완료 버튼 비활성화
- 완료 버튼 텍스트: `이벤트 참여 완료!`

**하단 브랜딩 문구:**
> "드림캐쳐스는 입주사 여러분의 양심과 신뢰를 믿습니다.  
> 남겨주시는 리뷰 한 줄은 더 좋은 공간을 만드는 유일한 자원이 됩니다."

---

### 4-5. 완료 화면 (각인 화면)

- 팡파르 애니메이션 (폭죽 또는 별 이펙트)
- 메시지:
  > "약속해주셔서 감사합니다!  
  > 냉장고에서 비타1000을 꺼내 드세요.  
  > 시원한 음료와 함께 성과 내는 하루 되세요."
- `[처음으로 돌아가기]` 버튼 (또는 10초 후 자동 복귀)
- 참여 번호 표시 (예: `참여번호 #0042`) — 심리적 완결감 제공

---

## 5. 데이터 모델

### 참여 기록 테이블 (`review_participations`)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | INTEGER PK | 자동 증가 |
| `participated_at` | DATETIME | 참여 일시 |
| `user_type` | ENUM('tenant', 'lounge') | 입주사 / 라운지 구분 |
| `naver_id` | VARCHAR(50) | 네이버 아이디 |
| `room_number` | VARCHAR(10) | 호실 번호 (입주사만) |
| `name_prefix` | VARCHAR(2) | 이름 앞 2글자 (라운지만) |
| `committed` | BOOLEAN | 약속 체크박스 동의 여부 |

## 6. 어뷰징 억제 로직

| 규칙 | 구현 방식 |
|------|-----------|
| 동일 네이버 ID 당일 중복 참여 차단 | `naver_id` + `participated_at(date)` 중복 체크 |
| 동일 호실 당일 중복 참여 차단 | `room_number` + `participated_at(date)` 중복 체크 |
| 중복 시 안내 메시지 출력 | "오늘 이미 참여하셨네요! 내일 또 만나요." |

## 8. 기술 스택 (권장)

| 영역 | 선택지 | 이유 |
|------|--------|------|
| 프론트엔드 | React + Tailwind CSS | 키오스크 UI 최적화 용이 |
| 백엔드 | Next.js API Routes | 단일 프로젝트 관리 |
| DB | SQLite (초기) → PostgreSQL (확장) | 저비용, 로컬 운영 가능 |
| 배포 | Vercel 또는 Raspberry Pi 로컬 서버 | 오프라인 환경 대응 가능 |
| 화면 자동 복귀 | 10초 타이머 후 시작화면 리셋 | 패드 전용 키오스크 모드 |
